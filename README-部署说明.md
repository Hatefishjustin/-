# 部署说明：EdgeOne Pages + 阿里云百炼 AI 深度解读

这个项目现在有两部分：

```
/index.html                前端页面（测评问卷 + 报告展示）
/node-functions/analyze.js  EdgeOne Pages Node Function（安全调用阿里云百炼 API）
```

## 为什么要多一个 node-functions/analyze.js？

大模型 API 都**不允许浏览器直接调用**——一是会暴露你的 API Key 给所有打开网页的人，
二是接口本身没有开放跨域（CORS），浏览器会直接拦截请求。

所以正确做法是：前端把数据发给你自己的一个小接口（`/analyze`），这个接口在服务器端
持有 Key，替你调用阿里云百炼，再把结果传回来。EdgeOne Pages 原生支持这种"函数文件"，
`node-functions/` 目录下的 `.js` 文件会被自动识别成 Serverless 函数，
文件名直接对应访问路径——`node-functions/analyze.js` 对应 `https://你的域名/analyze`。

## 部署步骤

### 1. 确认 Git 仓库结构

```
你的仓库/
  ├── index.html
  └── node-functions/
      └── analyze.js
```

### 2. 开通阿里云百炼，拿到 API Key

1. 打开 https://bailian.console.aliyun.com/
2. 登录阿里云账号（没有就注册，个人免费）
3. 首次开通会自动到账一次性 **7000万 Tokens** 免费额度（每个模型各100万，长期有效）
4. 左侧菜单点 **API-KEY 管理** → **创建新的 API-KEY**
5. 复制这个 Key（只显示一次，不要发给任何人或粘贴到聊天工具里）

### 3. 在 EdgeOne Pages 控制台连接仓库并部署

1. 打开腾讯云 EdgeOne 控制台 → **Pages**
2. 新建项目 → **导入 Git 仓库** → 选择你这个仓库，授权
3. 构建设置：因为是纯静态站点 + Node Functions，通常不需要额外的构建命令，
   保持默认（框架预设选 None / 无框架），EdgeOne 会自动识别 `node-functions/` 目录
4. 点部署

### 4. 配置环境变量（最关键的一步）

1. 进入你的 EdgeOne Pages 项目 → **设置** → **环境变量**（Environment Variables）
2. 新增一条：
   - **变量名**: `DASHSCOPE_API_KEY`
   - **变量值**: 你刚复制的阿里云百炼 API Key
3. 保存后，**重新触发一次部署**（环境变量的更新通常需要重新构建才会生效，
   在项目的"部署"页面找到重新部署/触发构建的按钮）

### 5. 完成

推送代码或手动重新部署后，测评做完会在结果页看到「AI 深度解读」标签页，
自动请求 `/analyze`，几秒到几十秒后展示通义千问生成的专业分析卡片。

## 关于费用

- 新用户开通百炼有一次性 **7000万 Tokens** 免费额度
- 当前使用的模型是 `qwen-plus`（比 `qwen-flash` 分析质量更高，用于生成有深度的心理解读），
  单独占用免费额度中的 100万 Tokens
- 一次分析大约消耗几千 Tokens（输入约4000-7000字原始作答记录 + 输出约1500字结构化分析），
  免费额度可以支撑很多次调用
- 代码里已加入同 IP 20 秒节流，防止被人反复点"重新生成"刷额度

## 关于国内访问速度

相比之前的 Vercel（`*.vercel.app` 域名在国内被污染，几乎打不开）和阿里云百炼
（相比调用国外接口更稳），这次的组合——EdgeOne 边缘节点 + 国内的阿里云百炼接口——
理论上应该能大幅改善国内访问和调用的速度与稳定性。

## 如果 AI 解读一直显示"请求失败"

大概率是这几种情况之一：

1. **环境变量没配对** —— 检查 EdgeOne 控制台的变量名必须精确是 `DASHSCOPE_API_KEY`
2. **配置后没有重新部署** —— 环境变量更新通常需要重新触发构建才生效
3. **阿里云百炼 Key 本身无效或额度用完** —— 去 bailian.console.aliyun.com 后台确认
4. **函数没有被正确识别** —— 确认文件路径精确是 `node-functions/analyze.js`
   （注意目录名是 `node-functions`，不是 `api`，这是与 Vercel 不同的地方）
5. **模型名过期或调整** —— 当前用的是 `qwen-plus`；如果阿里云未来调整了模型命名，
   需要同步修改 `node-functions/analyze.js` 里的 `model` 字段，最新模型列表见
   https://help.aliyun.com/zh/model-studio/models

## 关于本次的其他改动

- **MBTI题目已打散穿插**：不再单独成一个可辨认的"MBTI板块"，混在依恋模式题目之间，
  答题时看不出这几题在测什么
- **价值观算法已修正**：采用"曝光次数归一化"，每个价值观标签的得分会除以它在题库中
  的理论最高分再排序，避免题目设计导致的曝光次数偏差影响结果准确性
- **AI分析prompt已重写**：现在要求 AI 必须引用具体题目选择作为论据、挖掘选择之间的
  内在张力（比如依恋类型与爱语的错位），输出结构化 JSON 用于渲染成高级卡片UI，而不是
  一整段平铺直叙的文字
- **视觉已全面重做为苹果风**：近黑背景、单一强调色系统、大圆角卡片
- **新增导出功能**：结果页"导出报告"按钮，用浏览器原生打印能力生成 PDF，
  零外部依赖（不引入任何第三方截图库，避免重蹈"外部资源导致国内打不开"的问题）
